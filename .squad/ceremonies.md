# Ceremonies

## design-review
- **When:** before (condition: new feature involves UI or visual changes)
- **Facilitator:** Rosalina
- **Participants:** Luigi, Peach
- **Purpose:** Align on UX and visual approach before implementation begins

## retrospective
- **When:** manual
- **Facilitator:** Koopa
- **Participants:** Mario, Luigi, Bowser, Rosalina, Toad, Peach
- **Purpose:** Review completed work, surface friction, identify improvements

## planning
- **When:** manual
- **Facilitator:** Mario
- **Participants:** Peach, Yoshi, Luigi, Bowser
- **Purpose:** Prioritize and scope the next batch of work

## security-review
- **When:** before (condition: new feature exposes API endpoints, handles auth, or processes user input)
- **Facilitator:** Boo
- **Participants:** Bowser, Mario
- **Purpose:** Security review before shipping web-facing features
