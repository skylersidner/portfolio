# Implementation Logs

This directory holds living implementation notes for the portfolio project as development moves through each phase.

## Purpose

These logs are intended to capture meaningful progress and decision summaries without becoming overly granular. They should help explain:

- what was worked on
- what choices were made to move the work forward
- what risks, blockers, or follow-up items were identified
- how the application is taking shape over time

## Logging rules

- Keep entries concise and useful.
- Add new notes in reverse chronological order when practical.
- Each entry should include a timestamp down to the minute.
- Assume Central Time for all timestamps.
- Focus on decisions, progress, and outcomes rather than low-level implementation trivia.
- If a code-approach question needs deeper design input, consult the Architect.
- After each delivery phase, the Retro Analyst should add a short review with suggested workflow improvements.

## Suggested timestamp format

Use a heading like this:

## 2026-04-16 7:35 PM CT

## Files in this directory

- phase-journal.md — ongoing implementation summaries across phases
- decision-highlights.md — major implementation choices and why they were made
- retro-reviews.md — phase-end retrospective notes and workflow improvements
